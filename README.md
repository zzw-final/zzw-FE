## 🏹 기술스택 선정 이유

1. React

2. React-Query
- 레시피 사이트의 특성상 게시글마다 여러 장의 사진이 포함되어 있기 때문에 용량이 크다는 특징이 있습니다. 따라서 데이터를 여러번 호출한다면 속도 저하가 염려되었습니다.
- React-Query는 동일한 API 요청이 여러번 호출될 경우 한 번만 실행되고, 데이터가 stale해진 경우 적절한 시점에 알아서 업데이트된다는 점과 무한스크롤 구현, 로딩상태를 쉽게 관리 할 수 있다는 장점이 있어 도입하게 되었습니다.
- 또한 Global State와 Server State의 관심사를 분리함으로써 코드가 깔끔해지고 명확해진다는 점에서 많은 매력을 느꼈습니다.

3. browser-image-compression
- 서버에 전송되는 이미지의 크기를 1MB로 제한하기 위해 도입하였습니다.

4. React-Textarea-Autosize
- textarea의 높이를 텍스트에 따라 자동 조절해주는 라이브러리입니다. 채팅 기능을 구현하면서 텍스트의 길이에 따라 유동적인 UI를 그려내기 위해 사용하였습니다.

5. react-intersection-observer
- 무한 스크롤 기능 구현시 불러온 데이터의 마지막 위치를 알기 위해 사용하였습니다.

6. React-Cookie
- JWT 토큰을 쿠키에 안전하게 보관하기 위해 도입하였습니다.

7. Swiper
- 모바일 환경에서의 터치 이벤트에 잘 반응하는 Slider를 표현하기 위해 선택하였습니다.

8. Styled-components
- Styled-components는 JavaScript와 CSS 사이의 상수와 함수를 쉽게 공유 할 수 있습니다. 예를 들어, React에서는 props를 활용한 조건부 스타일링이 가능합니다.
- 또한 현재 사용중인 스타일만 DOM에 포함하는 장점이 있어 도입하였습니다.
- GlobalStyles 설정으로 스타일을 전역 관리하여 전체적인 UI에 통일성을 주었습니다.

9. Axios
- Interceptor를 통하여 서비스 흐름 중 유저 인증 플로우를 효과적으로 그리기 위해 선택하였습니다.

10. dayjs
- 레시피에 보여지는 Date를 상대적으로 표현하기 위해 사용하였습니다.

11. @stomp/stompjs
- stompjs는 WebSocket에서 메시지가 어떤 형식으로 사용될 지 지정해주고, Session 관리를 도와줍니다. 
- WebSocket에는 발행이나 구독같은 개념이 따로 없기 때문에 직접 설계해야 하지만, stomp에서는 발행/구독 메서드가 있어 편리하게 사용이 가능합니다.
- 또, Stomp를 이용하면 메시지에 Header를 줄 수 있기 때문에 Header를 기반한 인증 처리가 가능하기 때문에 도입하였습니다.
